Project Questions

  1. Question: On average, how many requests can `ab` complete in 10 seconds with all the power of two concurrency levels between 1 and 256?

    Answer: Average was approximately 1,433 requests in 10 seconds across all tested concurrency levels.

  2. Question: Why are there diminishing returns at higher concurrency levels?

    Answer: It happens due to resource limitations and increased overhead. As the number of concurrent requests increases, the server may reach its processing capacity, network bandwidth may become saturated, and context switching overhead may increase, leading to less efficient processing of additional concurrent requests.

  3. Question: What's the performance difference when requesting HTTP and HTTPS?

    Answer: In my test, HTTPS requests were about 60% slower than HTTP requests. For example, at a concurrency level of 64, HTTP achieved 6,973 requests/10s, while HTTPS achieved 2,601 requests/10s. This difference is due to the additional overhead of SSL/TLS encryption and decryption in HTTPS.

  4. Question: How can GitHub respond so quickly?

    Answer: Because it serves static content through a global CDN. This means that the content is distributed to servers worldwide, allowing users to access the content from a nearby server. Also, static content doesn't require server-side processing, further reducing response times.

  5. Question: What is your site's "Total Blocking Time" (TBT) according to PageSpeed Insights?

    Answer: I observed TBT to be 0 ms! This is because the page is static, with minimal JavaScript, resulting in no significant main thread blocking.
